Detailed Description

The Generator class is one of the cores of this program.

This class is superclass of the Element class and stores a sequence of UpdatePairs which represents the process how the data is generated. By using subclasses (the GenDict and GenList class), one can group the Element instances in one Generator and control all the updating process.

void Generator::merge_update_chain ( const Vec< UpdatePair > &  new_chain)
protected

Merge a sequence of UpdatePairs to update_chain of this instance.

Parameters
constVec<UpdatePair> &new_chain).
Returns
None.

In merging a sequence of UpdatePairs, relative orders of the sequence passed by the new_chain and the update_chain are retained so that dependency order of the updating process will not be broken. Duplication of UpdatePair will be removed. If there is no explicit preference, elements in the new_chain will be appended to the end of update_chain. This method is thread-unsafe because it accesses the update_chain thread-globally. Ensure this method is called from (via) Python (such as append_updater of the Element class) and not called in a multithreads context.

Vec<UpdatePair> Generator::update_chain
protected

Sequence of UpdatePairs. It represents the process how the data is updated (filled by keys and values).

This sequence of UpdatePairs is executed one by one in the hello method. In this updating process, the Element instances (the first element of UpdatePair) uses the Updater instances (the second element of UpdatePair) to update its data. An element of this sequence is added in append_updater method of the Element class. In that method, update_chain stored in the external Generator in the Updater instance will be merged to the update_chain of this instance.
